# Helm 产品架构

**产品名**：Helm  
**定位**：单人公司 × AI 团队 —— 一家公司定义清楚，AI 替你干活，你只做决策。  
**目标用户**：单个创业者、小团队主理人、AI 原生公司创始人  
**核心承诺**：5 分钟内从「建公司 + 提第一个需求」到「看到 AI 开始执行」

**关联文档**：[技术架构](./opc_technical_architecture.md)、[CompanySpec](./opc_companyspec_integrated.md)、[AgentSpec](./opc_agentspec_integrated.md)

---

## 一句话价值

> **你当 CEO，AI 当团队。先定义公司是谁、干什么、为谁干；再提需求，COO 替你拆任务、调团队、跑执行。**

---

## 核心实体：Company（公司）

**Helm = One Person Company**，公司是第一实体。一切需求、任务、Agent 都归属一家公司。

**Company Spec（公司规格）** — 创建公司时：

| 用户动作 | 填写项 | 说明 |
|----------|--------|------|
| **选模板** | 0 项 | 目标、服务对象、产出、质量均由模板预填，秒开 |
| **新建空白** | 最少 3 项 | 名字、干什么（目标）、为谁干（服务对象）|

**最少 3 项**：公司名、目标、服务对象。其余由模板预填或渐进披露。

**约束**：所有需求必须服务于公司目标；COO 按 Company Spec 拆解任务，Agent 执行时自动对齐目标与受众。

---

## 心智模型

```
     [ 公司 Company ]  ← 核心：目标 + 服务对象
           │
     [ 你 = CEO ]
           │
           │  唯一通道：在公司边界内提需求、做决策、验收结果
           ▼
     [ COO = AI 总调度 ]
           │
           ├── Agent A ──► 完成任务
           ├── Agent B ──► 完成任务
           └── Agent C ──► 完成任务
```

**四个原则**：
1. **公司先行** — 先建公司、定目标与服务对象，再提需求
2. **你只管 COO** — 不跟具体 Agent 打交道
3. **COO 管一切** — 拆解需求、分配任务、调度协作、上报问题
4. **任务流可见** — 谁在干什么、卡在哪，一眼看清

---

## 用户体验闭环

1. **提需求** — 在公司内描述想要什么（自然语言）
2. **看方案** — COO 生成「任务流程图」，你审批
3. **AI 开干** — 审批通过后，Agent 按图执行
4. **收结果** — 完成时通知，交付物一目了然（文件、链接、PR、报告）
5. **不满意？** — 反馈给 COO，自动回到步骤 2

**用户不需要知道**：AgentSpec、CheckState、Adapter、DAG……这些藏在产品背后。

---

## 魔法时刻：5 分钟

- **0 min**：选模板（0 填写）或新建（填 3 项：名字、目标、服务对象）
- **1 min**：在公司内提第一个需求
- **2 min**：COO 生成任务流，你点「同意」
- **3 min**：看到第一个 Agent 开始干活
- **5 min**：首个可交付物出现

---

## 治理与边界

**预算**：设「本月最多花多少」，超了就停。**审批**：新招 Agent、大方案变更，需你点头。**随时叫停**：任何任务、Agent 可暂停或取消。

**一眼看懂**：打开产品即可回答——谁在干什么、花了多少钱、卡在哪。

**做**：需求→任务流→审批→执行→验收；单一决策入口；预算硬顶；开箱模板；本地优先；问题显式。  
**不做**：通用聊天；Jira/GitHub 替代；企业 RBAC；多租户 SaaS；技术术语外泄。
